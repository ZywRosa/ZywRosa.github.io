3.处理返回操作

  1. Activity 栈管理：
    • 使用 startActivityForResult 和 onActivityResult 方法（适用于 Android 11 及以下）。
    • 或者使用 ActivityResultContracts 和 ActivityResultLauncher（适用于 Android 12 及以上）。
  2. Fragment 栈管理：
    • 使用 FragmentTransaction 来管理 Fragment 的添加、替换和回退操作。
    • 确保在回退时能够正确显示之前的 Fragment。
  3. 返回键处理：
    • 在详情页面的 onBackPressed 方法中处理返回操作，可以选择性地添加一些确认逻辑（例如弹出对话框）。

4.实现步骤

  1. 创建项目并设置基础布局：
    • 创建一个新的 Android 项目。
    • 在 activity_main.xml 中设计主界面的布局，包括 Toolbar、FrameLayout 和 BottomNavigationView。
  2. 创建 Fragment：
    • 为聊天、联系人、位置、设置分别创建四个 Fragment。
    • 在聊天 Fragment 中实现 RecyclerView 的布局和 Adapter。
  3. 实现 Tab 栏跳转：
    • 在 MainActivity 中设置 BottomNavigationView 的点击事件监听器，根据点击的 Tab 切换相应的 Fragment。
  4. 完善 RecyclerView：
    • 在聊天 Fragment 中初始化 RecyclerView 和 Adapter。
    • 为 Adapter 中的 ViewHolder 设置点击事件监听器。
  5. 实现详情页面跳转和返回操作：
    • 在点击事件中创建 Intent 并跳转到详情页面。
    • 在详情页面中处理返回操作，确保能够正确返回到聊天 Fragment。
  6. 测试和优化：
    • 测试各个界面的跳转和返回功能。
    • 优化 UI 布局和交互效果。
